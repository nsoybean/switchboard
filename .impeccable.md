## Design Context

### Users
Developers and non-coders who orchestrate multiple AI coding agents (Claude Code, Codex, Bash) in parallel. They're monitoring several sessions simultaneously, switching between agents, reviewing diffs and terminal output. The tool needs to serve both power users running complex multi-agent workflows and less technical users who need clarity without noise.

### Brand Personality
Quiet, neat, powerful, fast. Switchboard doesn't announce itself — it recedes so the work stays in focus. Every element is purposeful. The interface communicates competence through restraint, not through visual spectacle.

### Emotional Goals
- **Control**: The user should feel they have complete situational awareness across all running agents
- **Calm**: Dense information presented without visual stress — no competing accents, no decorative clutter
- **Speed**: The interface should feel instant and lightweight, like a native tool that's always been on the machine

### Aesthetic Direction
Utilitarian dev tool in the lineage of Superconductor — information-dense, neutral, monospace-forward. Not brutalist (too raw), not luxury (too precious). Think: well-maintained control room where every indicator means something. Light and dark modes, warm-tinted neutrals (current OKLCH palette with hue ~49-56 is correct). No flashy accents — status colors (green/amber/blue/red) are the only chromatic elements, used strictly for semantic meaning.

### Anti-References
- Dashboard-style UIs with big cards, gradient accents, and decorative charts
- "AI product" aesthetics: cyan-on-dark, purple gradients, glowing borders
- Over-designed dev tools that prioritize visual novelty over information density

### Design Principles
1. **Information density over decoration** — Every pixel should convey state, structure, or content. If an element is decorative, remove it.
2. **Quiet until relevant** — Status colors, indicators, and alerts appear only when they carry meaning. The default state is neutral and calm.
3. **Monospace as native language** — JetBrains Mono is the primary voice. Inter supports where proportional text improves scannability (labels, descriptions). Don't fight the monospace identity.
4. **Consistent spatial rhythm** — Tight, purposeful spacing. Use the 4pt scale. Group related elements tightly, separate sections generously. No uniform padding everywhere.
5. **Native-feeling, not web-feeling** — Should feel like a compiled desktop app, not a web page in a frame. Minimal border-radius, no bouncy animations, no decorative shadows.
